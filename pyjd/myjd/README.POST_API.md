## POST requests to the API

All POST requests to the API MUST include this payload as JSON:

```json
{
  "apiVer": 1,
  "url": "/relative/path/to/the/endpoint",
  "params": "<array>",
  "rid": 12345
}
```

`apiVer` MUST be an integer. Current version is 1.

`url` MUST be an string. It's relative path of the request url, AKA the endpoint. ex: `/my/connect`

`rid` is the id of the current request. It's an random number generated by the client.
MUST be an integer, MUST be unique for each request and SHOULD increase after every request.
Responses will also include a `rid` value and the client SHOULD validate that it matches the request id.

`Params` MUST be an array. For most endpoints is the array of the values for each option that the endpoint accepts.
The order of the values MUST be the same order as they are defined on the API spec, ex:

```json
{ "params": [1, true, "youtube.com", false] }
```

For endpoints that have an explicit QUERY type param defined on the API spec (ex: `/addLinks`), the query MUST be serialized as a map _inside_ the array, ex:

```json
{
  "params": [
    {
      "configInterface": null,
      "defaultValues": true,
      "description": true,
      "enumInfo": true,
      "includeExtensions": true,
      "values": true,
      "pattern": ""
    }
  ]
}
```

Additionally, POST requests going thought MyJD (AKA, all of them except connections via local deprecated API)
MUST encrypt the entire JSON payload following the algoritm of the `/my/connect` endpoint.
